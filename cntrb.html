Contribute to the caprice libraries

All contributors will be listed in the README file for the respective caprice library they contribute to. You may find other contributors in the IRC channel (#kf-engine@irc.freenode.net). Questions, patches or concerns can be directed in that channel.

Bugs

Bug fixes are always welcome, patches can be attached directly on github under the bug tracker.

Correctness

Contributors of unit tests which help in verifying the correctness of various facilites provided in the caprice libraries will be greatly appreciated. It's a project goal to provide 100% unit test coverage to verify that all functionality is correct in conformance with the C and C++ standards.

Documentation

Contributors can also help document various facilities, and improve the documentation page on this site. Documentation is a project goal not only does it allow developers who may be intersted in contributing to this project, it also helps lowering our bus factor.

Ports

Developers looking to port the libraries to other architectures are in need as well. Our design goal is to have full support for most mobile, desktop, and embedded architrectures.

Performance, Benchmarking, Analysis

Benchmarkers are welcome to analyse the performance of various facilities in the caprice libraries, and are encouraged to send reports to all developers if something is a notable bottleneck or not performing like it should, or could be otherwise.
